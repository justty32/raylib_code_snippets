# 2D 程序式動作編寫手冊（給 AI agent 與人類）

本手冊說明 `snippets_cpp/procedural_anim_2d.cpp` 這套 2D 骨架動畫系統，目的是讓
**任何 AI agent**（或你自己）能依據文字需求，產生或調整角色「動作」。

把這份文件整份貼給 agent 當 context，再加上你的需求（例如「做一個上勾拳，蓄力久一點、收勢帶點抖動」），agent 就有足夠資訊輸出可用的結果。

> 系統的「真理來源」是程式碼本身（`procedural_anim_2d.cpp`，含繁體中文注解）。本手冊與程式碼衝突時，以程式碼為準；改動系統後請同步更新本手冊。

---

## 0. 兩種動作表示法

同一套播放引擎（彈簧 + idle 疊加 + FK）可以被兩種來源驅動，agent 兩種都可能要產生：

| 表示法 | 是什麼 | 何時用 | 需重編譯？ |
|--------|--------|--------|-----------|
| **A. 程序式動作函式（C++）** | 在 `author_action()` 裡用數學曲線描述一個招式 | 純演算法、參數化、可程式控制的動作 | 是 |
| **B. 關鍵幀動作資料檔（文字）** | 在時間軸上設幾個關鍵幀，引擎內插＋彈簧補出中間 | 配合動作編輯器；「設關鍵幀＋下 prompt 補幀」 | 否（熱載入） |

兩者最終都產出同樣的東西：**每根骨頭相對 rest 的角度偏移（弧度）** + **根節點位移目標**。引擎再疊上彈簧（次級動作）與 idle，跑 FK 畫出來。

> 表示法 B 的檔案格式是「編輯器（Phase 2）將採用」的規格草案，見第 5 節。先照此格式產生 clip，編輯器完成後即可直接載入。

## 1. 座標系統與慣例

- **Y 軸向下**（raylib 慣例）。角度單位在資料/手冊裡都用**度(°)**，程式內部用弧度。
- 角度方向：`0° = 右(+x，即「前方」)`、`90° = 下`、`-90°/270° = 上`、`180° = 左`。
- 角色**側面朝向**。`facing = +1` 面向右(+x)；`facing = -1` 在 X 軸鏡射，**同一份動作數值左右通用**，agent 一律以「面向 +x（前方）」來思考即可。
- 每根骨頭的角度是 **local**（相對父骨）。世界角度 = 父骨世界角度 + local。

---

## 2. 骨架規格（17 根骨頭）

骨頭以「近側(N, near＝靠鏡頭、畫在前)」「遠側(F, far＝畫在後、顏色較暗)」區分左右肢。
索引順序固定（見 `enum`），rest 為休息姿勢的 local 角度：

| 索引 | 名稱 | 父骨 | rest local° | rest world°(面向右) | 長度 | 說明 |
|----:|------|------|----:|----:|----:|------|
| 0 | `HUB` | — | 0 | 0 | 0 | 骨盆／根，不可見 |
| 1 | `SPINE` | HUB | -90 | -90 | 46 | 脊椎（向上） |
| 2 | `CHEST` | SPINE | 0 | -90 | 34 | 胸 |
| 3 | `NECK` | CHEST | 0 | -90 | 10 | 頸 |
| 4 | `HEAD` | NECK | 0 | -90 | 26 | 頭 |
| 5 | `FUARM` | CHEST | 188 | 98 | 32 | 遠側上臂（向下略偏後） |
| 6 | `FFARM` | FUARM | -8 | 90 | 28 | 遠側前臂 |
| 7 | `FHAND` | FFARM | -4 | 86 | 12 | 遠側手 |
| 8 | `NUARM` | CHEST | 172 | 82 | 32 | 近側上臂（向下略偏前） |
| 9 | `NFARM` | NUARM | -8 | 74 | 28 | 近側前臂 |
| 10 | `NHAND` | NFARM | -4 | 70 | 12 | 近側手 |
| 11 | `FTHIGH` | HUB | 92 | 92 | 44 | 遠側大腿 |
| 12 | `FSHIN` | FTHIGH | -4 | 88 | 42 | 遠側小腿 |
| 13 | `FFOOT` | FSHIN | -84 | 4 | 16 | 遠側腳（朝前） |
| 14 | `NTHIGH` | HUB | 88 | 88 | 44 | 近側大腿 |
| 15 | `NSHIN` | NTHIGH | -4 | 84 | 42 | 近側小腿 |
| 16 | `NFOOT` | NSHIN | -84 | 0 | 16 | 近側腳（朝前） |

主要出招肢體用**近側(N)**；遠側(F)通常做反向擺動以維持平衡與層次感。

## 3. 正負號慣例（最重要，照這個寫才不會反向）

偏移量是「加在 rest 上」的角度。記住每個部位「正偏移往哪轉」：

| 部位 | `+` 偏移 | `−` 偏移 |
|------|---------|---------|
| `SPINE` / `CHEST` / `HEAD` | 軀幹上半**往前(+x)傾** | 往後仰 |
| 近/遠側**上臂**（`*UARM`） | 手臂**往後 / 往上**揮 | 往**前 / 往上**揮（出拳即負） |
| 近/遠側**前臂**（`*FARM`） | — | 負值＝手肘**屈曲**（手往肩收）；正值＝伸直 |
| 近/遠側**大腿**（`*THIGH`） | 腿往**後**擺 | 腿往**前 / 往上**抬（前踢即負） |
| 近/遠側**小腿**（`*SHIN`） | 屈膝（小腿往後） | 小腿往前彈出 |

口訣：**手往前打、腿往前踢，都是「負偏移」**；軀幹**前傾為正**。

根節點位移 `root`：`x` = 面向方向的**前方為正**（被擊退就用負）；`y` = **上為正**（彈跳）。引擎會自動乘上 `facing` 並換算螢幕座標。

## 4. 表示法 A：程序式動作函式（C++）

### 4.1 可用的塑形積木

```cpp
clamp01(x)            // 夾在 [0,1]
ease_out_cubic(x)     // 快→慢（出招爆發、收尾平緩）
ease_in_cubic(x)      // 慢→快（蓄力、加速）
bell(x)               // 0→1→0，一次起伏（衝刺、整體強調）
strike_drive(u, wind, hit)
    // 帶正負號的「蓄力→打擊→收勢」驅動量：
    //   u<wind          : 由 0 緩出到 −1（蓄力）
    //   wind<=u<hit      : 由 −1 急速到 +1（出招瞬間）
    //   u>=hit           : 由 +1 緩出回 0（收勢）
```

### 4.2 一個動作長什麼樣

在 `author_action(act, u, off, root)` 的 `switch` 裡新增一個 `case`。輸入是正規化相位
`u∈[0,1]`，輸出是把 `off[<骨頭>]`（弧度，乘 `D = kD2R`）與 `root`（前方 x、上 y）填好。
**你只描述「想要的姿勢曲線」，彈簧會自動加上延遲與過衝**，不要自己手刻回彈。

範例：以 `strike_drive` 把一個純量散到多根骨頭（出拳）：
```cpp
case ACT_PUNCH: {
    const float s   = strike_drive(u, 0.30f, 0.55f);          // 0~0.30 蓄力，0.55 出拳
    const float fwd = std::max(0.0f, s), back = std::max(0.0f, -s);
    off[NUARM] = (-95 * fwd + 18 * back) * D;   // 出拳：上臂往前；蓄力：往後
    off[NFARM] = (  8 * fwd - 55 * back) * D;   // 伸直 / 蓄力屈肘
    off[SPINE] = (  6 * fwd -  5 * back) * D;   // 前傾 / 後仰
    root.x     =  26 * fwd;                      // 出拳瞬間前衝
    break;
}
```

### 4.3 新增一個招式的步驟

1. 在 `enum Action { ... }` 加 `ACT_XXX`。
2. 在 `kDur[]` 對應位置加它的秒數（如 `0.5f`）。
3. 在 `act_name()` 加顯示名。
4. 在 `author_action()` 的 `switch` 加 `case ACT_XXX:` 並填 `off[]` / `root`。
5. 在 `main()` 綁一個按鍵 `if (IsKeyPressed(KEY_X)) trigger(ACT_XXX);`（可選），
   或加進自動展示的 `seq[]`。

### 4.4 調校指南（手感）

- **幅度(°)**：四肢動作 30–95° 較明顯；軀幹 5–15°；過大會穿模。
- **時間切點**：`strike_drive` 的 `wind`/`hit` 控制「蓄力多久、何時出招」。蓄力久＝有威脅感；`hit` 越接近 `wind`＝出招越突然。
- **彈簧 K/D**（在 `main` 的 `off_spring[i].step(target, K, D, dt)`）：
  - 預設 `K=240, D=20` → 略低於臨界阻尼，帶一點過衝（有手感）。
  - 想更「Q彈／抖動」：降 `D`（如 12–16）。想更「俐落不晃」：升 `D`（接近 `2*sqrt(K)≈31`）。
  - 升 `K` → 追得更快更硬。
- **根運動**：`root.x` 前衝/擊退；demo 會彈回原位（正式遊戲改成累加到角色座標即可）。

## 5. 表示法 B：關鍵幀動作資料檔（編輯器格式 v1 草案）

純文字、無外部相依、人與 agent 都好讀。一個檔 = 一個動作 clip。
時間用**正規化 0..1**（乘上 `duration` 得秒數）。`#` 為註解。骨頭名稱見第 2 節。

```
# move clip v1
name        uppercut
duration    0.55          # 秒
facing      relative      # 數值以「面向 +x」撰寫，編輯器負責鏡射
loop        false
spring      240 20        # 疊加層彈簧 K D（0 0 = 關閉次級動作）

# 關鍵幀： key <t 0..1> <到下一幀的緩動>
#   緩動可用： linear | ease_in | ease_out | ease_in_out
#   縮排行： "<BONE> <deg>"（相對 rest 的偏移）或 "root <x_前> <y_上>"
key 0.00 ease_out
  root 0 0
key 0.28 ease_in            # 蓄力底
  NUARM 20
  NFARM -50
  SPINE -6
key 0.50 ease_out           # 出招
  NUARM -70
  NFARM -10
  SPINE 8
  root 18 0
key 1.00 linear             # 收勢回 rest（未列的骨頭自動回 0）
  root 0 0
```

**內插規則**（編輯器將實作）：每根骨頭的偏移在相鄰兩關鍵幀之間，用該段的緩動曲線
內插（未在某幀列出的骨頭視為偏移 0）。算出每幀的「目標姿勢」後，再交給**同一組彈簧**
產生次級動作 —— 所以即使只有 3、4 個關鍵幀，動作也會有延遲與過衝的生命感。

> 注意：表示法 A 與 B 的差別只在「目標偏移從哪來」（程式算 vs 內插關鍵幀）。
> 彈簧、idle 疊加、FK、鏡射、根運動全部共用。

## 6. 給 AI agent 的工作流程

**情境一：從文字需求生成新動作**
1. 判斷該用 A（要程式化／參數化）或 B（要在編輯器時間軸上微調）。預設用 **B**（資料檔，免重編譯）。
2. 依需求選關鍵時間點，照第 3 節正負號填關鍵幀；軀幹小幅、四肢大幅。
3. 用 `ease_in` 表蓄力、`ease_out` 表出招爆發後的緩和。
4. 需要抖動/Q感 → 調低 `spring` 的 D；需要俐落 → 調高 D。
5. 輸出完整 clip 檔（或 A 的 `case` 程式碼片段）。

**情境二：在既有關鍵幀之間「補幀／調參數」（編輯器主流程）**
1. 讀入使用者現有的 clip（已有的關鍵幀）與這段的 prompt（例如「0.3 到 0.6 之間多一個收手的預備」）。
2. 在指定時間區間**插入新的 key**或**修改既有 key 的偏移/緩動**，不要破壞使用者已鎖定的關鍵幀數值（除非被要求）。
3. 保持 `t` 嚴格遞增、`0` 與 `1` 兩端存在、未列骨頭即回 rest。
4. 只輸出修改後的完整 clip，方便編輯器熱載入比對。

**情境三：調手感**
- 只動 `spring K D`、各段 `ease_*`、或整體幅度縮放，不動關鍵幀的時間結構。

## 7. 注意事項 / 常見錯誤

- **方向搞反**：90% 的錯誤來自正負號。動手前先回看第 3 節。「手往前打、腿往前踢 = 負」。
- **rest 一定要有效**：偏移是「相對 rest」；把所有偏移設 0 必須是合理的站姿。
- **別手刻回彈**：收勢只要把偏移收回接近 0，過衝交給彈簧。自己再加抖動會打架。
- **幅度別爆**：上臂超過約 ±120°、軀幹超過約 ±25° 容易穿模或鬼畜。
- **彈簧穩定**：`step()` 已做次步進，K 很大時仍穩；但 K 配上太低的 D 會持續振盪不收斂。
- **時間軸**：clip 的 `t` 用 0..1；秒數由 `duration` 換算。`key` 必須按 `t` 遞增排列。
- **左右**：一律以面向 +x 撰寫，靠 `facing` 鏡射，不要為了左右各寫一份。
